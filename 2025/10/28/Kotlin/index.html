函数的可见性修饰符

java中有public，private，protected和default（什么都不写）这4种函数使用哪种修饰符，Kotlin中也有4种，分别public，private，protected和internal，需要使用哪种修饰符时，直接定义在fun关键字的面前即可

修饰符 访问范围 适用场景(函数)
public(默认) 全局可见：同一模块，跨模块，子类，外部文件均可访问 通用工具函数，对外暴露的API（如SDK接口）
internal 模块内可见：同一模块捏的所以文件访问，跨模块不可见 模块内部复用的函数（避免暴露不必要的接口，减少跨模块依赖）
protected 类内+子类可见：仅当前类，其子类(含镍一模块/跨模块子类)可访问，外部不可见 类的核心逻辑函数，允许子类重写或复用，但不允许外部直接调用
private 私有可见：仅当前作用域(类/文件/代码块)内可访问 类内部辅助函数（仅当前适用），文件内局部复用函数（不对外暴露）

分场景详细说明：

1.顶层函数（文件直接定义的函数）

顶层文件不属于任何类，其可见性由修饰符直接控制，核心规则

public(默认)：全局可见，跨文件、跨模块均可调用

internal：仅当前模块内的所有文件可调用，跨模块找不到该函数

private：仅当前文件内可调用，同一模块的其他文件不可见

protected：不适用于顶层函数（编译报错），因为顶层函数没有父类，protected 的「子类可见」语义无意义

2.成员函数（类/接口内的函数）

成员函数隶属于类或接口，修饰符作用于[类的访问权限]，核心规则：

public(默认)：类的外部、子类、跨模块均可调用

internal：仅当前模块内的任意代码可调用（无论是否是该类的实例）

protected：仅当前类内部、其子类（含跨模块子类）可调用，类的外部不可见

private：仅当前类内部可调用，子类、外部均不可见

与java可见性修饰符的区别

Kotlin修饰符 java对应的修饰符 核心差异
public public 行为一致(全局可见)，但kotlin默认是public(java默认是包内可见)
internal internal kotlin按[模块]隔离，java按[包]隔离，范围不同(模块可包含多个包)
protected protected kotlin中protected仅子类可见(不包含同一包内非子类)，java包含同一包内类
private private kotlin中private可用于顶层函数(文件内可见)，java无顶层函数

数据类与单例类

数据类：

数据类通常占据着非常重要的角色，它们用于将服务器或数据库中的数据映射到内存中，为编程逻辑提供数据模型的支持

数据类通常需要重写equals(),hashCode(),toString()这几个方法，其中，equals()方法用于判断两个数据类是否相等。hashCode()方法作为equals()的配套方法，也需要一起重写，否则会导致HashMap，HashSet等hash相关的系统类无法正常工作。toString方法用于提供更清晰的输入日志，否则一个数据类默认打印出来的就是一行内存地址

filterNot(predicate) 保留不满足条件的元素（与 filter 相反） list.filterNot { it.name.isEmpty() }（筛选非空名称）
filterIsInstance<T>() 保留集合中类型为 T 的元素 list.filterIsInstance<String>()（筛选所有字符串元素）
filterNotNull() 保留非空元素（用于可空类型集合） list.filterNotNull()（过滤掉 null）
distinct() 去除重复元素（基于 equals 判断） listOf(1,2,2,3).distinct() → [1,2,3]
distinctBy(selector) 按指定规则去重（selector 结果相同则视为重复） list.distinctBy { it.name }（按名称去重）
drop(n) 丢弃前 n 个元素，返回剩余元素 listOf(1,2,3,4).drop(2) → [3,4]
dropLast(n) 丢弃后 n 个元素 listOf(1,2,3,4).dropLast(2) → [1,2]
take(n) 取前 n 个元素 listOf(1,2,3,4).take(2) → [1,2]
takeLast(n) 取后 n 个元素 listOf(1,2,3,4).takeLast(2) → [3,4]
slice(indices) 取指定索引范围的元素（indices 为 IntRange 或集合） listOf(1,2,3,4).slice(1..2) → [2,3]

将集合中的元素转换为另一种类型，返回新集合。

函数 功能说明 示例代码

flatMapIndexed(transform) 展平转换时携带索引 list.flatMapIndexed { idx, it -> listOf(idx, it) }
flatten() 将嵌套集合直接展平（等价于 flatMap { it }） listOf(listOf(1,2), listOf(3,4)).flatten() → [1,2,3,4]

sumByDouble(selector) 转换后求双精度和（返回 Double） list.sumByDouble { it.price }（求总价格）
average() 求平均值（适用于数值类型） listOf(1,2,3).average() → 2.0
maxByOrNull(selector) 按 selector 转换后求最大值元素（返回可空类型，无元素时为 null） list.maxByOrNull { it.age }（找年龄最大的元素）
minByOrNull(selector) 按 selector 转换后求最小值元素 list.minByOrNull { it.price }（找价格最低的元素）

reduce(operation) 从第一个元素开始，累加执行 operation（无初始值，至少需 1 个元素） listOf(1,2,3).reduce { acc, it -> acc + it } → 6（累加）

空指针检查

空指针的本质是：访问了一个值为 null 的变量的属性、方法或索引，导致程序直接崩溃

程序员最长出现的问题

可空类型系统

Kotlin却非常科学的解决了这个问题，它利用编译时判断空检查的机制几乎杜绝了空指针异常，虽然编译时判空检查的机制有时候会导致代码变得比较难写，但是不用担心，Kotlin提供了一系列的辅助工具，让我们能轻松地处理各种判空情况

我们向doStudy( )函数传入一个null参数，会提示报错。kotlin将空指针异常的检查提前到了编译时期，如果我们的程序存在空指针异常的风险，那么在编译的时候会直接报错，修正之后才能成功运行，这样就可以保证程序运行时期不会出现空指针异常了

这里你可能会有疑惑，所有的参数和变量都不可能为空？这里我们就可以引出可为空的类型系统

可为空的类型系统

可为空类型是什么样的，很简单，就是在类名的后面加上一个问号。比如 Int 表示不可为空的整形，而 Int？就表示可为空的整形；String 表示不可为空的字符串，而 String？就表示可为空的字符串

这样我们就解决了空指针异常的问题

判空辅助工具

?.

首先我们先来学习最常用的 ?. 操作符，这个操作符非常的好理解，就是当对象不为空时正常调用响应的方法，当对象为空时则什么都不做

?:

我们学习另一个非常常用的 ?. 操作符。这个操作符的左右两边都接收一个表达式，如果左边表达式的结构不为空就返回左边表达式结果，否则就返回右边表达式的结果

我们通过一个具体的例子来结合 ?. 和 ?: 这两个操作符，来深刻理解一下

!!

Kotlin的空指针检查机制并非那么智能，有的时候我们可能从逻辑上已经将空指针异常处理了，但Kotlin的编译器并不知道，这个时候它还会编译失败

这里我们定义了一个可为空的全局变量content，然后子啊main( ) 函数里先进行一次判空操作，当content不为空的时候才会调用printUpperCase( ) 函数，在printUpperCase( )函数里。我们将content转换为大写模式，最后打印出来，看上去没什么问题，但是很遗憾，这段代码一定是无法正常使用的，因为printUpperCase( )函数并不知道外部已经对content变量进行了非空检查，在调用toUpperCase( )方法时，还认为这里存在空指针风险，从而无法编译通过

在这种情况下，我们如果想编译**，可以使用非空断言工具**，写法是在对象的后面加上 !! 意思是告诉Kotlin 我非常确信这里的对象不为空，不用来空指针检查了

这个非空断言工具要谨慎使用，能有更好的方法就用别的方法
